# Build Week scope and provenance

JobAuto existed as a personal automation before the public Studio work. The
Build Week effort created the generic candidate boundary, the visual local
application and the public distribution path.

The OpenAI Build Week submission period began July 13, 2026. The checkpoints
below and the subsequent public-release commits distinguish the pre-existing
personal workflow from the product work completed during that period.

## Build Week chronology

- **July 15-17:** isolated the earlier personal workflow and introduced generic
  candidate, evidence, adaptation and submission contracts.
- **July 17-19:** built Studio onboarding, `.tex` preservation, PDF-content
  import, file-free profiles, discovery, campaigns, document supervision and
  the Codex Chrome handoff contract.
- **July 18-19:** captured the cross-domain real-agent evidence packs, including
  the five-application non-IT campaign and hash-verified sandbox receipts.
- **July 19-20:** added the applications dashboard, comparable ATS readiness,
  before/after PDF inspection, live progress and cancellation, plugin/runtime
  preflight, packaged demo replay, privacy audits and clean-install validation.

The public repository is exported with clean history and generic contributor
metadata. This chronology describes the meaningful extension while avoiding
publication of the earlier personal repository history.

The public commit dates establish when the sanitized release was published;
they do not independently prove the day-by-day chronology above. The submitted
Codex `/feedback` Session ID and the captured GPT-5.6 traces are the supporting
Build Week records. This document is a scope disclosure, not a substitute for
those records.

The public branch removes the personal profile, application history, private
templates and user-specific runtime modules from the distributable product. It
keeps synthetic examples only.

## Added or generalized during Build Week

- candidate-owned profile and evidence contracts;
- exact `.tex` source import, PDF-content import and reviewed generated layouts;
- section-level adaptation controls and project/skill policies;
- candidate-isolated prompts and artifact stores;
- Codex web discovery from saved search preferences;
- generic campaign filtering, deduplication and tracking;
- ATS brief, document agents, reviewer and repair orchestration;
- real PDF compilation, parsing, hashes and layout metrics;
- local Studio interface and run telemetry;
- idempotent campaign-level Chrome handoff queue, sandbox verification, real-receipt tracking and tracker synchronization;
- public release audit, synthetic fixtures, wheel packaging and documentation.

This file records product provenance. It does not claim that an actual
third-party Chrome submission is performed inside the public FastAPI process.
